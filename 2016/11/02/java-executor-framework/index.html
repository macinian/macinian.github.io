Java Executor Framework 相关知识梳理

1. 从Executor接口谈起

Java执行器框架围绕着Executor接口而展开，Executor接口将任务的提交和执行解耦，被提交至执行器的任务，会在将来的某个时候得到执行，具体执行的行为根据执行器的具体实现而定。Executor接口形式简单、抽象层次高，包含两个重要元素：Runnable接口所代表的任务对象，以及Executor接口所代表的执行器对象。从任务和执行器两条线分别进行拓展，Java执行器框架均可以从中获得更多的特性。

2. 任务篇

2.1 Runnable的定义和缺失

Runnable接口设计了一套通用协议给那些希望当处于活跃状态时去执行一些代码的对象，例如其最知名的实现类Thread，活跃状态指的是一个线程对象被start之后和stop之前，此时该线程的run方法期望得到执行。
通常我们会写一个类实现Runnable接口，并在创建一个新线程时将其实例传递给其构造方法，当我们启动该线程的时候，这个实现了Runnable接口的类的run方法就可以在一个独立的线程中得到执行。

那么任务被提交之后呢？执行是否完成、执行的结果是什么、不需要执行了可以取消吗？ Runnable接口的缺失主要在于无法跟踪任务的执行情况和管理任务的生命周期。

2.2 Future和RunnableFuture

Future代表一次异步计算的结果。Future接口定义了可超时阻塞的get方法用于获取异步计算的结果，可以通过cancel方法取消一个任务，并提供了相应的方法检查一个任务是否完成、是否取消等。Future接口使得一次异步计算任务可跟踪、可管理、可获取结果，有效填补了Runnable的不足。RunnableFuture顾名思义，就是集Runnable和Future于一身的接口。

2.3 如你所想的FutureTask

FutureTask代表一个支持取消的异步计算任务，实现了RunnableFuture接口，也就是说它可以把自身提交给执行器去异步执行(Runnable)，同时还可以通过自己的实例方法来跟踪这次执行的状态、获取计算结果、执行取消操作等(Future)。

FutureTask需要实现一个最基础的功能，就是当异步任务未执行完成时，调用future.get方法的线程需要阻塞等待，一旦该任务执行完成则所有相应阻塞的线程都需要被及时唤醒。更一般化的来看，基本上就是实现一个阻塞锁的逻辑，多个线程去竞争一个资源/判断一个状态，不成功则阻塞等待，一般会设计一个队列来持有这些被阻塞线程的引用；当资源被释放/状态变更时，则唤醒一个或多个阻塞的线程。
AQS（AbstractQueuedSynchronizer）作为一个阻塞锁及同步器实现的通用框架，设计了一个CLH队列的变体数据结构（变成了双向队列）来提供支持。FutureTask(JDK8)采用了一种更轻量级的方案，利用了一种lock-free的算法/数据结构: Treiber Stack Algorithm。Treiber Stack算法使用简单的CAS加自旋的策略来更新栈顶元素，从而实现了一种线程安全的并发栈。

FutureTask利用Treiber Stack作为线程等待栈（是栈或者队列在这里区分的意义不大）持有因调用future.get方法而阻塞的线程引用。在任务未执行结束时调用future.get方法会通过awaitDone方法，将当前线程阻塞并构造一个WaitNode持有线程引用，作为栈顶元素入Treiber Stack。而在任务执行结束的时候（正常完成、异常、取消时），会调用finishCompletion方法唤醒所有相应的阻塞线程。

FutureTask实现同时用到了自旋和阻塞这两种技术，任何互斥协议都提出了同一个问题：如果你无法获得锁，你该怎么办？通常有两种处理方式，第一种是不断重试，这类锁就是自旋锁，不断重试的动作就是自旋或者忙等待；另一种是把自己挂起，让操作系统调度有需要的别的线程，这种方式称为阻塞。当持有锁的时间比较短的时候适合使用自旋锁，反之可能阻塞锁相对开销更少，一般也会组合两者，自旋一段时间之后再进入阻塞。FutureTask的Treiber Stack实现使用了自旋策略，而利用Treiber Stack作为工具实现线程等待，则使用了阻塞的方式。

另外FutureTask有一个volatile代码细节，FutureTask构造器注释ensure visibility of callable，多个变量均需要保障可见性，利用happens-before规则，把最后写的那个声明为volatile即可。

2.4 持续可拓展

Java Executor Framework提供的可拓展性并没有止步于FutureTask。FutureTask应当被看作异步计算任务的一种默认实现，开发者以此为基础或参照，可以设计出性能更加、特性更丰富的实现方案。例如guava通过对FutureTask简单的继承，就拓展出一种可监听的异步计算任务ListenableFutureTask，当计算完成的时候，指定的多个监听器将得到通知回调，提供了一种更优雅的编写任务执行代码的模式。
